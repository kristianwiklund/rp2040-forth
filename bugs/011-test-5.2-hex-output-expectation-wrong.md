# Bug 011: Test 5.2 — `HEX 1a . DEC` expected value is wrong (test suite bug)

## Test

**5.2** — `HEX 1a . DEC`
Got: `1a`  Expected: `26`

## Summary

The firmware output `1a` is **correct**.

The command sequence executes left-to-right:
1. `HEX` — switches base to 16
2. `1a` — parsed as hexadecimal 26, pushed onto stack
3. `.` — prints TOS in the **current base** (still 16) → prints `1a`
4. `DEC` — switches base back to 10 (after printing)

Test 5.1 (`HEX ff . DEC` → `ff`) demonstrates the same behaviour and passes.
The note on test 5.2 says "output in DEC" but `DEC` is called **after** `.`,
so it cannot affect the output.

## Fix

Either:

- Change the expected output to `"1a "` (what the firmware actually produces), **or**
- Rewrite the input as `HEX 1a DEC .` (switch to DEC before printing) and keep
  the expected output `"26 "`.

## Severity

Test suite error only — no firmware change needed.
