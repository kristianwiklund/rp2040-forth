# Bug 010: Test 2.3 — `SWAP` test expectation is wrong (test suite bug)

## Test

**2.3** — `1 2 SWAP . .`
Got: `1 2`  Expected: `2 1`

## Summary

The firmware's `SWAP` produces `1 2`, which is **correct** standard Forth
behaviour.

Standard Forth: `SWAP ( x1 x2 -- x2 x1 )`.

- Before: x1=1 (NOS), x2=2 (TOS)
- After: x2=2 (NOS), x1=1 (TOS)
- `. .` prints TOS first: `1`, then `2` → `1 2`

The test expectation `2 1` is inconsistent with the standard and with the rest
of the test suite.  The IF/THEN/ELSE tests (6.1–6.7) all pass, and they rely
on `SWAP` working correctly (`: THEN HERE SWAP ! ;`). A `SWAP` that put `2`
back on top would leave hole addresses unpatched and break those tests.

## Fix

Change the expected output in `test/cases/stack.py` test 2.3 from `"2 1 "` to
`"1 2 "`. The note should read: `"SWAP: old NOS=1 is now TOS, printed first"`.

## Severity

Test suite error only — no firmware change needed.
